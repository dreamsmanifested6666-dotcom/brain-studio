# v1.0 — Real-fMRI activation pipeline

Status snapshot at the v1.0 ship of the real-fMRI integration.
The brain visualizations across most of the site are now driven by
real Neurosynth meta-analysis data (Yarkoni 2011, CC0) projected
onto the HCP-MMP-360 parcellation (Glasser 2016, CC0 via HCP).
The 20-region literature-informed `signaturePatterns` in
`lib/regions.ts` is retained as the fallback for idle states and
for pages that haven't been wired yet.

## What ships (PR-A through PR-G)

### Pipeline geometry

- `frontend/public/parcellation/hcp_mmp_left.json` —
  10,242 ints, fsaverage5 left hemisphere parcel ID per vertex.
- `frontend/public/parcellation/hcp_mmp_right.json` — same
  for right hemisphere.
- `frontend/public/parcellation/parcel-labels.json` — 360
  parcels with canonical Glasser name + Cole-Anticevic network.
- `frontend/content/parcellation/region-to-parcel-map.json` —
  hand-curated alias from the site's 20 RegionId vocabulary to
  groups of HCP-MMP parcels.

### Build scripts (run once, commit outputs)

- `backend/scripts/build_parcellation.py` — fetches the ENIGMA
  fsaverage5 projection of Glasser-2016 and the Cole-Anticevic
  parcel-name LabelKey; writes the three parcellation JSONs.
- `backend/scripts/build_neurosynth_activations.py` — NiMARE
  MKDA-Chi2 meta-analyses on Neurosynth-v7 across the stimulus
  map; projects each z-map to fsaverage5 via nilearn's
  vol_to_surf, aggregates into HCP-MMP-360 parcels, composites
  per editorial decomposition, sigmoid-squashes to [0, 1],
  emits JSON. 53 stimulus files committed (~500 KB total).

### Frontend infrastructure

- `frontend/store/useBrainStageStore.ts` — new
  `parcelActivations` field + `setParcelActivations` setter.
- `frontend/components/brain/BrainAnatomy.tsx` — per-vertex
  parcel coloring path. Falls back to the 20-region path when
  `parcelActivations` is empty.
- `frontend/components/brain/BrainActivationDriver.tsx` — thin
  client component that pushes a parcel map on mount and
  resets on unmount.
- `frontend/components/brain/BridgesActivationScroller.tsx` —
  IntersectionObserver-driven activation scroller, scoped to
  `data-bridge-section` elements (Bridges).
- `frontend/components/brain/PassageActivationScroller.tsx` —
  generalised IntersectionObserver scroller over
  `data-activation-id` elements (Faust, Dante, future).
- `frontend/components/brain/ProvenanceFooter.tsx` — inline
  methodology + citation chip with expanding
  "not a measurement of your brain" disclaimer.
- `frontend/lib/loadActivations.ts` — client-side typed fetch
  helpers per category.
- `frontend/lib/loadActivationsServer.ts` — server-side
  filesystem loader (used at SSR/build time).

## What does NOT change

- Cellular page intentionally hides the persistent brain.
- The TRIBE-as-model-class explanation on home + about stays —
  it accurately describes the model class TRIBE belongs to.
- Mirror room still serves real predictions from the existing
  Render-deployed BGE-small backend; the 20-region projection
  there is unchanged.
- Editorial disclaimers, three-language discipline, and the
  "the model is not the mind" framing are preserved
  everywhere — and made more prominent via ProvenanceFooter.

## What's deferred (v1.0.1 / v1.1)

### PR-E — Tours timeline activations

The Tours room renders a sequence of scenes per tour; the
ideal wiring is per-scene timeline frames so the brain animates
through the tour's narration. The stimulus map in
`build_neurosynth_activations.py` does not yet include tour
stimuli — adding them requires:

1. Define tour scene compositions (editorial work; each scene
   has a few terms).
2. Re-run `build_neurosynth_activations.py` with the tour
   stimuli added (the existing term z-map cache will be reused
   for any already-computed terms).
3. Extend the Tours autoplayer to push parcel activations per
   scene with linear interpolation between frames.

The Tours room is currently still driven by `setActivations`
calls against the 20-region vocabulary (the existing autoplayer
code).

### PR-F — Mirror backend HCP-MMP-360 projection

The Mirror room's Render-deployed BGE-small backend currently
returns 20-region activations only. To return parcel-level
activations as well, the backend needs:

1. A new `backend/tribe/region_mapping_hcpmmp.py` mirroring the
   existing `region_mapping_destrieux.py` pattern — lazy
   nilearn parcellation load (HCP-MMP via the same projection
   the frontend uses), disk-cached vertex assignments.
2. Updates to `predict.py` to emit both 20-region and
   360-parcel maps in the response.
3. A Render redeploy via `git push` (the existing
   `render.yaml` blueprint).

When this lands the Mirror visualization will shift from
20-region blobs to 360-parcel detail underneath, matching the
rest of the site's granularity.

## How to regenerate the data

```bash
cd brain-studio
backend/.venv-tribe/Scripts/python.exe backend/scripts/build_parcellation.py
backend/.venv-tribe/Scripts/python.exe backend/scripts/build_neurosynth_activations.py
# Outputs are written to frontend/public/parcellation/ and
# frontend/public/activations/. Commit and deploy.
```

The Neurosynth-v7 dataset (~50 MB) and the term-level z-map
cache (~30 MB across 30 terms) are kept under
`backend/data/{neurosynth,term_zmaps}/` and excluded from git
via `backend/.gitignore`. First run downloads the dataset; reruns
reuse the cache.

## Editorial discipline (what stayed protected)

The user's brief was explicit: "Do not let the new system erode
any of [the methodology disclaimers, the three-language
discipline, the predictions-are-not-measurements framing]." None
of those eroded. Every brain visualization on every wired room
now carries an inline `ProvenanceFooter` that names the term
composition, the data source (Neurosynth), the parcellation
(HCP-MMP-360), the citation (Yarkoni 2011), the license (CC0),
and an expanding "not a measurement of any individual brain"
disclaimer.
